# 产品原则

这个 App 的目标不是完整记录每一手牌，而是在真实线下现金局里，快速抓住值得复盘的关键手牌。

## 一句话定位

线下现金局关键手牌速记工具：快速记录大底池、纠结牌、对手读牌和行动路线，方便之后复盘。

## 核心用户场景

- 用户在手机上快速打开 App。
- 一手牌结束后，用最少点击记录关键信息。
- 当场只记录值得复盘的牌，不追求完整 session 流水账。
- 之后可以回到待复盘列表，继续补充行动路线、公共牌、复盘三问，并导出 Markdown。

## 核心体验原则

1. 克制优先

   每个页面只保留最重要的动作。不要为了“功能完整”堆按钮。

2. 工具化展示

   首页只展示当前状态和可执行动作，不放产品口号、目标说明或使用引导。用户打开就是为了记录，不是为了读说明。

3. 速记优先

   输入应该尽量点选完成，例如手牌、公共牌、位置、动作、常见下注尺度。文本输入只用于备注和必要的自定义。

4. 行动顺序优先

   行动路线的主交互是“行动顺序”，而不是让用户手动拼一条路线。用户按位置顺序点动作，系统生成路线文本。

5. 默认自定义优先

   翻前结构先不展示 Hero open、面对 open、3bet pot 等模板。默认自定义，直接进入行动顺序记录，减少现场理解成本。

6. 允许漏记后修正

   线下实战一定会漏记，所以路线必须支持插入、编辑、删除动作，并在修改后重新计算底池、当前下注、待跟和下一位行动人。

7. 防止重复动作

   一条街没有新的 bet / raise 时，所有应行动玩家完成一轮动作后，本街应闭合，不再循环 check。若有 bet / raise，只让仍需回应的人继续行动。

8. 防止回忆错误

   补路线时，Hero 手牌、当前公共牌、当前街和底池应常驻顶部，减少用户反复回看和记错。

9. 中文可理解

   面向中文用户，位置展示应包含中文名称，例如“枪口位（UTG）”“大盲位（BB）”。

## 当前功能边界

当前只考虑：

- 线下现金局
- 4–10 人桌
- straddle
- bb / 实际筹码金额切换
- Hero 手牌点选
- Flop / Turn / River 公共牌点选
- 行动路线速记、插入、修正
- 主页手牌记录左滑删除，并二次确认
- 待复盘列表
- 回放与复盘三问
- Markdown 导出
- 本机本地存储
- Session 结束、历史重新进入与继续记录

当前明确不做：

- 线上牌局接入
- 登录账号
- 云同步
- 约局、俱乐部、支付、结算
- 实时策略建议或 solver 推荐
- HUD、对手数据库
- 图片导出
- 复杂备份系统

## 数据原则

数据只保存在 iPhone App 本机本地。

- 关闭 App 不丢。
- 手机重启不丢。
- 正常更新 App 通常不丢。
- 卸载 App 时，iOS 会按默认逻辑清除 App 沙盒，本地数据会丢。

当前阶段接受这个边界，不做账号、云同步或自动备份。

## 迭代纪律

为了避免产品和代码变成一坨无法维护的东西，每次迭代必须同步维护文档：

- 改产品体验或交互：更新 `docs/PRODUCT.md` 或 `docs/DECISIONS.md`。
- 改用户可见功能：更新 `docs/CHANGELOG.md`。
- 做了重要取舍：记录到 `docs/DECISIONS.md`。
- 每次交付前至少完成一次构建验证。
