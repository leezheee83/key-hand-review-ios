## 2026-07-11｜更新手牌复盘标签

- `为什么记这手？` 改为 `标记这手`。
- 标签更新为：`待复盘`、`诈唬`、`抓诈`、`Hero Fold`、`价值`、`冤家`、`自定义`。
- 移除早期标签：`大池量`、`纠结`、`对手读牌`。
- 历史旧标签会归一展示为 `待复盘`。
- 待复盘列表改为待补全优先、最新在前，不再按 `纠结` 排序。

## 2026-07-11｜历史 Session 支持继续记录

- 历史场次重新进入后，如果本场已结束，会显示 `继续记录本场`。
- 点击后该 session 恢复为进行中，并重新显示 `＋ 记关键牌`。
- 新手牌会继续归入这个历史 session。
- `返回场次列表` 仍保留，用于只查看历史、不恢复记录。
- 更新关键决策：结束本场是软归档，不是永久锁定。

## 2026-07-11｜简化翻前结构与手牌删除

- 翻前结构区域默认只展示 `自定义`，不再展示 Hero open、面对 open、limped / ISO、3bet pot 等模板。
- 主页 `手牌记录` 改用 iOS 原生列表交互，支持左滑露出红色删除按钮。
- 点击删除后弹出确认框，确认后删除对应手牌记录。
- 更新关键决策：减少翻前模板理解成本；删除动作保持二次确认。

## 2026-07-11｜首页克制展示

- 删除主页说明性文案：`本次核心目标`、`只抓住关键手牌`、`大池量、纠结牌、对手读牌；其他牌不打扰。`
- 删除主页使用引导：`一手结束后打开，先保存，再补完整路线。`
- 主页场次卡片仅保留场次信息、记录数量、状态和操作按钮。
- 空状态文案改为 `暂无手牌记录。`

## 2026-07-11｜本地 Session 历史与固定公共牌信息条

对应提交：`5194273 Add local session history and sticky board context`

- 路线页顶部新增固定信息条，常驻显示 Hero 手牌、公共牌、当前街和底池。

## 2026-07-11｜修复行动闭合后循环 check

对应提交：`dccdb7d Stop closed streets from cycling actions`

- 一条街没有新的 bet / raise 时，所有应行动玩家完成动作后，本街闭合。
- 有 bet / raise 时，只推荐仍需回应的玩家。
- 已经不欠动作的位置置灰，避免重复 check / call。
- 漏记时仍可在本街路线中插入或编辑动作，并重新计算行动状态。

## 2026-07-11｜行动顺序成为路线记录主入口

对应提交：`884e62d Streamline action-order route entry`

- `行动顺序与记录` 成为本街动作录入主入口。
- `本街路线` 收敛为生成文本和修正入口。
- 支持插入、编辑、删除动作。
- 修正后重新计算底池、当前下注、待跟和下一位行动人。
- Call 默认使用系统计算的待跟金额。
- Open / bet / raise / 3bet / 4bet 使用“下注到 X”的语义。

## 2026-07-11｜行动顺序时间线编辑
